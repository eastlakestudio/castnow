# Walkthrough - iOS App Media Upgrade (v2.3.0)

CastNow Pro iOS 应用已完成媒体功能重大升级，现在支持多路流并发采集。

## 主要变更内容

### 1. 并发流支持 (同时传 Screen & Camera)
- **重构逻辑**：修改了 `_startBroadcast`，现在支持同时捕获屏幕共享和摄像头，并将它们合并到一个 `MediaStream` 中发送。
- **Web 适配**：保持与 Web 端接收器的逻辑对齐，确保接收端可以识别到两路视频轨道并进行 PIP 或并列显示。

### 2. 麦克风默认静音
- **隐私保护**：根据您的要求，广播启动时麦克风状态默认为 `Mute`（静音）。
- **初始化**：`_isMuted` 初始值设为 `true`，且在流采集时立即应用 `track.enabled = false`。

### 3. 源选择增强与默认值
- **默认勾选**：进入广播界面时，屏幕、摄像头和麦克风默认均为勾选状态。
- **强制校验**：在启动前会检查是否至少选择了一个视频源（屏幕或相机），防止发送空视频流。

### 4. 状态同步
- **控制台更新**：工具栏中的麦克风图标会根据初始静音状态显示为红色（Mic Off），点击后可正常解除静音。

## 验证结论
- [x] **多流并发**：已验证代码逻辑支持将屏幕、相机轨道并发注入 PeerJS 单次呼叫。
- [x] **默认静音**：`_isMuted` 在初始化和采集阶段均生效。
- [x] **归档完成**：本实施方案已同步归档至 `dev_process/implementation_plan_v2.3.0.md`。
