# Walkthrough - Receiver UI Refinement (v2.4.0)

本版本对接收端的 UI 进行了优化，特别是针对“单流（仅屏幕或仅摄像头）”场景下的自动化处理。

## 修改内容

### 1. 自动隐藏布局控制
- **优化**：当接收端仅探测到一路视频流时，底部的“分屏/画中画切换”和“画面交换”按钮将**自动隐藏**。
- **逻辑**：只有在“屏幕”和“摄像头”同时在线时，这些多流调度按钮才会显示。

### 2. 满视图显示优化
- **改动**：在单流模式下，移除了所有多流相关的容器占位（Stream A/B），直接通过 `remoteVideo` 全屏自适应显示唯一的视频源。
- **效果**：用户无需手动切换，系统会自动根据流的数量决定是显示“单流满速模式”还是“双流协作模式”。

## 验证建议
- [x] **单流验证**：使用手机或 PC 发送单一视频流，确认底部控制栏仅保留“静音/全屏/麦克风”，且画面自动填满容器。
- [x] **双流验证**：同时发送屏幕和摄像头，确认布局切换按钮重新出现，且各项功能正常。
