# CastNow Checkpoint

## 当前状态 (Current Status)

1. **基础功能与 UI 恢复**：
   - 投屏配置界面左上角已恢复**返回按钮**，支持点击返回首页。
   - 恢复了 **RTMP 模式** 选项及推流地址和密钥输入表单。
   - 修复了因为 `AppLocalizations` 缺失引入和 `capabilities.codecs` 空安全检查导致的编译报错，当前项目可以正常编译通过。

2. **1710 / 3420 分辨率花屏及模糊问题**：
   - **模糊问题**：此前尝试通过将宽度缩放至固定 `1280` 来对齐 Stride，但由于高分屏下缩放系数过大（达到 2.6x 以上），导致接收端画面非常模糊、文字无法辨认。
   - **花屏问题**：虽尝试在握手建立后调用 `setCodecPreferences` 优先协商 VP8 编解码器（企图绕过苹果 VideoToolbox 的 H.264 Stride 对齐 Bug），但从实际效果看，接收端依然存在斜向拉伸花屏。这说明 VP8 优先协商可能未在底层生效，或者 capturer 物理采集的纹理本身已带有 stride 偏移。

---

## 下一步计划 (Next Steps)

1. **移除缩放逻辑**：
   - 按照您的要求，**停止进行强制缩放**，将发送端的 `scaleResolutionDownBy` 恢复为原画 `1.0`（或清理相关缩放代码），避免画面模糊。
2. **彻底排查花屏根源**：
   - 排查 PeerDart 底层协商 SDP 时为何没有成功切换至 VP8 编码。
   - 寻找无需强制缩放即可解决 macOS 原生采集 Stride 花屏的更彻底方法。
