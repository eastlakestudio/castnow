# CastNow 产品升级需求文档 (PRD)

## 1. 文档概述

本文档旨在定义 CastNow 系列应用的功能升级，核心目标是支持多源流（摄像头、麦克风、系统桌面）的同步传输，并在接收端提供高度灵活的 UI 布局管理。

---

## 2. 核心功能升级

### 2.1 发送端 (Cast 端) - 多源同步传输

发送端用户可以根据需要选择不同的媒体资源进行叠加或独立传输。

- **多源并发**：支持摄像头 (Camera)、麦克风 (Microphone) 和系统桌面 (Screen/App Desktop) 同时进行采集与传输。
- **资源可选性**：
  - 用户可以在开始投屏前或投屏中，动态勾选/取消勾选传输源。
  - 例如：仅传屏幕、传屏幕+摄像头、传屏幕+麦克风、或三者同时传输。

### 2.2 接收端 (Receive 端) - 灵活布局管理

接收端在接收到多路流（如屏幕流与摄像头流）时，支持用户自定义显示方式。

- **多样化布局模式**：
  - **并列模式 (Side-by-Side)**：两个窗口等大或按比例并排显示，不重叠。
  - **画中画模式 (Picture-in-Picture, PiP)**：主次分明，一个小窗口悬浮在另一个全屏窗口之上。
- **灵活交互逻辑**：
  - **层级切换 (Swap)**：在画中画模式下，一键切换哪个是“大窗口”哪个是“小窗口”（例如从“屏幕为主，摄像头为辅”切换到“摄像头为主，屏幕为辅”）。
  - **位置拖拽 (Draggable)**：在画中画模式下，用户可以自由拖动小窗口在界面上的显示位置（如：左上角、右下角、或任意位置）。
  - **动态层级关系**：支持通过点击或菜单灵活改变 Camera 和 Screen 的上下层级关系。

---

## 3. 用户场景示例

1. **在线网课/演示**：老师一边演示手机 APP 桌面，一边在右下角通过摄像头显示自己的讲解画面，且可以随时根据需要把摄像头画面放大到全屏。
2. **移动游戏直播**：主播同时传输游戏画面与自拍画面，互动时可灵活拖动自拍画面位置以防遮挡游戏关键 UI。

---

## 4. 技术要求初步预研

- 需确保同步：音频、视频、桌面流在传输与播放过程中需保持高度同步。
- 低延迟：针对多路流的合并编解码需进行优化，确保实时性。
- UI 实现：接收端需实现自定义的可拖拽 Overlay 组件，支持容器层级 (Z-index) 的动态调整。
